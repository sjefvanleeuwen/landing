From "Space Invaders" to "GPT Galaxy Quest": A Comprehensive Journey of Collaborative Game Development

In one of my most exciting projects to date, I partnered with ChatGPT to build an HTML5 canvas game that evolved from a simple Space Invaders clone into a dynamic, Galaxian-inspired shooter—"GPT Galaxy Quest.".

What began as a straightforward coding request quickly turned into an iterative, collaborative process filled with creative challenges, incremental improvements, and insightful design decisions. In this article, I’ll share the complete story of how we developed this game step by step, including my example prompts and key code excerpts that highlight our evolution from concept to a full-featured game.

1. The Beginning: A Basic Space Invaders Clone

My Prompt: "create space invaders in HTML canvas"

Our journey started with a classic challenge: develop a Space Invaders-style game using the HTML5 canvas. ChatGPT responded with a clean, basic implementation featuring a player-controlled rectangle, a grid of aliens, and rudimentary collision detection. This initial code laid the groundwork for further enhancements. Although the game was simple, it provided a solid base upon which to build more complex dynamics.

2. Refining Alien Movement

After reviewing the initial code, I noticed an issue: when aliens reached the right edge, instead of moving down one row and reversing direction, they simply fell off the screen. I prompted:

My Prompt: "when the aliens arrive at the right of the screen they dont go one position down and scroll to the left. instead they fall down until they are off the screen."

ChatGPT revised the alien movement logic to detect the screen edge and trigger a single downward move before reversing horizontal movement. This change was subtle but essential for mimicking classic arcade behavior. The improvement ensured the alien formation maintained a cohesive, predictable pattern while still challenging the player.

3. Transforming into a Galaxian-Style Game

With the movement fixed, I wanted to add more flair and dynamic behavior:

My Prompt: "cool, now turn it into galaxian"

At this stage, the game evolved into a Galaxian-inspired shooter. Instead of static aliens, our new design featured a formation that gently oscillated horizontally. Most importantly, aliens were given the ability to break formation and dive toward the player. This introduced a new state system—each alien could be in either a "formation" state or a "diving" state. ChatGPT explained:

"When an alien is selected to dive, it leaves its formation and moves downwards while gradually steering toward the player."

This shift added an element of unpredictability and intensity, setting the stage for even more advanced behavior.

4. Adding Counterattacks: Dive and Shoot

To up the ante, I challenged ChatGPT further:

My Prompt: "make it dive shoot back at you"

We enhanced the game by giving diving aliens the ability to shoot enemy bullets at the player. The diving aliens now not only advanced toward the player but also fired projectiles, forcing the player to dodge both the aliens and their bullets. ChatGPT introduced an enemy bullet system complete with vector calculations that allowed bullets to be aimed directly at the player’s current position.

5. Enhancing Attack Patterns and Shooting Frequency
Eager to refine the experience even further, I issued the next prompt:
My Prompt: "make it shoot earlier and more bullets, and start with 1 attacker then 2, then 3 and repeat that."
Here, we implemented several improvements:
  • Earlier Shooting: Diving aliens were modified to shoot their first burst after only 500 milliseconds.
  • Burst Fire: Each shooting event now fires a burst of three bullets (with a slight angular spread) instead of a single bullet.
  • Attack Cycle: We introduced a cycling mechanism where the game starts with 1 attacker, then increases to 2, then 3, before repeating the cycle.
These enhancements significantly increased the game’s intensity and visual excitement. The burst fire code snippet above played a key role in achieving this dynamic behavior.
6. Balancing the Game: Smart Shooting Threshold
I wanted to ensure the game remained fair, so I provided further feedback:
My Prompt: "to improve gameplay do not let the aliens shoot when they are too low."
ChatGPT introduced a shooting threshold (set at 70% of the canvas height), so that diving aliens would stop shooting when they got too close to the bottom of the screen. This adjustment maintained balance by preventing the player from being overwhelmed by enemy fire when the enemies were already in close proximity.
7. Adding Score, Leveling, and Difficulty Scaling
Next, I aimed to create a complete game experience with progression:
My Prompt: "when all the aliens are shot reset. also show score, increase the difficulty."
The game was updated to:
  • Track the Score: Each alien destroyed added 10 points to the score.
  • Level Up: When all aliens were eliminated, the game reset the formation, increased the level, and ramped up difficulty.
  • Difficulty Scaling: With each new level, the dive speed, enemy bullet speed, and dive interval were adjusted to provide a progressively challenging experience.
This progression system transformed the game from a static challenge into a dynamic, evolving experience that rewarded players for their skill.
8. Polishing the User Experience: Start Screen and Game Over
A professional game needs a polished user interface, so I prompted:
My Prompt: "great now start the game displaying GPT Galaxy Quest and press enter to start the game when the player loses, display game over and show press enter for a new game, reset the score."
We added:
  • A Splash Screen: Displaying “GPT Galaxy Quest” with instructions to press Enter to start.
  • A Game Over Screen: Displaying a “Game Over” message, the final score, and a prompt to press Enter for a new game.
  • State Management: The game state was managed with distinct modes—"start", "playing", and "gameover"—to control the flow and reset the game cleanly.
