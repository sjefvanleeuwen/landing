Building an AI Sales Agent Prototype for Configure‑Price‑Quote — My MIT Course Project

Created on 2025-03-18 08:08

“Imagine a world where an AI sales assistant instantly understands customer needs and delivers tailored EV charging recommendations in seconds.”

As part of my coursework at MIT, I set out to build a proof‑of‑concept AI agent that functions as a Configure‑Price‑Quote (CPQ) sales assistant for a fictitious electric vehicle (EV) charging solutions provider. This prototype demonstrates how modern language models, combined with lightweight tooling, can transform the way sales teams engage customers — even in complex technical domains.

Why an AI‑Powered Sales Agent?

“Sales conversations should focus on customer needs, not data lookups.”

In today’s fast‑moving EV market, customers demand fast, accurate answers about product specifications, installation requirements, and pricing comparisons. Manually searching spreadsheets or static catalogs slows down response times and risks inconsistency. My goal was to show how an AI agent can serve as a first‑line sales consultant, freeing human teams to focus on relationship building and closing deals.

The Prototype at a Glance

This is a prototype — a working command‑line tool built with JavaScript, LangChain, and Ollama — designed to illustrate core CPQ capabilities:

  • ✅ Product Discovery: Provide detailed specs for home, commercial, and fast‑charge EV chargers

  • ✅ Comparisons: Highlight differences between models (e.g., budget vs. enterprise offerings)

  • ✅ Recommendations: Suggest optimal solutions based on customer use case and electrical setup

  • ✅ Contextual Memory: Maintain conversation context so follow‑up questions feel natural

“Think of it as a virtual sales rep that never sleeps — always ready to answer the next question.”

How It Works (High Level)

  1. Structured Product Catalog All fictional EV charger models are stored in a JSON‑style data source, complete with power ratings, pricing, and feature details.

  2. Natural Language Understanding User queries trigger intent detection: product lookup, comparison, or recommendation. LangChain routes each request to the appropriate chain.

  3. Dynamic Prompt Engineering For comparisons, the agent constructs a side‑by‑side product summary prompt so the underlying language model can generate a clear, concise analysis.

  4. Interactive CLI Interface Users interact via a simple command‑line interface: ask questions, compare chargers, or request installation guidance.

AI‑powered CPQ tools can drastically reduce time‑to‑quote, improve quote accuracy, and enhance customer experience. While my MIT project remains a prototype using a fictitious product catalog, it demonstrates a clear path toward scalable sales automation — from simple CLI demos to integrated CRM plugins or web‑based chat assistants.

I’m excited to iterate further: integrating live pricing feeds, vehicle compatibility checks, and omnichannel deployment.
