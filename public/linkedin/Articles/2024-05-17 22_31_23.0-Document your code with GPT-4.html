Introducing Code-Narrator: Simplifying Documentation with GPT-4

Documentation is a critical aspect of any sizeable project, yet it often poses a significant challenge for developers. The advent of ChatGPT has revolutionized this process, providing an innovative solution that streamlines the creation of comprehensive documentation. After extensive experimentation, it was discovered that ChatGPT can generate high-quality documentation for code files, surpassing the quality of manual efforts. This realization led to the development of Code-Narrator, a client designed to simplify the documentation process.

What is Code-Narrator?

Code-Narrator is a self-documenting tool that leverages GPT-4 to analyze code files and automatically generate documentation. This tool is language-agnostic and has been tested with various programming languages such as TypeScript, GraphQL, Solidity, C#, Kotlin, and more. As long as the files are in plain text, Code-Narrator can work its magic seamlessly.

Transforming Developers from Writers to Editors

The primary goal of Code-Narrator is to simplify the documentation process for developers, transforming them from writers to editors. While developers are still responsible for verifying the accuracy of the generated documentation, the initial writing is handled by the AI. If GPT-4 produces incorrect documentation, it often indicates that the code needs clarification or that a brief comment should be added to guide the AI. Essentially, if GPT-4 cannot comprehend the code, it may be too complex for the next developer.

Key Features and Limitations

  • Token Limit Constraint: Code-Narrator is constrained by GPT-4’s 8192-token limit, which can be problematic for extensive code files. Those with access to the gpt-4-32k variant should expect better results, as the quality of the generated documentation heavily depends on the length of the input (question + file content).

  • Configuration: Upon its first run, Code-Narrator creates a configuration file by analyzing the project and prompts for review and accuracy. Key configuration aspects include the “include,” “config_files,” and “source_path” settings.

  • Documentation Generation: In its second run, Code-Narrator generates documentation for the entire project. The process takes approximately 45 minutes to complete from scratch but is highly flexible, allowing for custom pages such as How-To guides, tutorials, FAQs, READMEs, and more.

Supported Features

  • Multilingual: Code-Narrator supports 25+ languages (as many as GPT-4 supports).

  • Versatile Formats: Capable of generating documentation in various formats (LaTeX, HTML, with Markdown being the default).

  • Custom Content Creation: Easily create How-To guides, tutorials, FAQs, READMEs, and other bespoke content.

Cost Efficiency

Code-Narrator consists of 45 files and 1712 lines of code, with the total cost of generating documentation for the entire project being approximately $2.5. This represents a significant saving compared to manual documentation.

Benefits Observed During Development

  • Improved Function Naming: Reading the generated documentation can lead to revising vague or overly general function names, resulting in better code and documentation.

  • Concise Input: Focusing on reducing the size of the input file (liquid) becomes a fun challenge, helping to optimize the process.

  • Enjoyable Interaction: Working with ChatGPT proves more enjoyable than manually writing documentation.

  • Minimal Input for Tutorials: It is pleasantly surprising how little input is required to create How-To guides and tutorials using GPT-4.

  • Encouragement to Refactor: The 8K token limitation may be restrictive, but it also encourages developers to refactor their code for improved readability and structure.

Future Improvements

As a prototype and proof of concept, Code-Narrator has some limitations. Future improvements and features planned include:

  • Documenting only diffs in files.

  • Building a static website using Jekyll or Docusaurus.

  • Adding unit tests for the project.

  • Generating comprehensive documentation for unit tests.

  • Enhancing the generation of Mermaid charts using GPT-4.

  • Implementing better caching to reduce costs.
